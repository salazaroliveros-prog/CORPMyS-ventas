📋 Paso a Paso: Despliegue en GitHub Pages

Paso 1: Preparar archivos localmente

  1. Abre terminal en la carpeta "11. paginaweb"
  2. Verifica que tengas todos los archivos:
    ls -la index.html ls -la stitch_pageweb/ ls -la *.html
  3. Crea archivo .gitignore (opcional pero recomendado):
    # .gitignore node_modules/ .DS_Store *.log .env

Paso 2: Inicializar repositorio Git

  1. En terminal, ejecuta:
    git init git add . git commit -m "Initial commit: Aplicación web integrada"

Paso 3: Crear repositorio en GitHub

  1. Ve a github.com y crea una cuenta (si no tienes)
  2. Haz clic en "New repository"
  3. Nombre del repositorio: mi-pagina-web (o similar)
  4. IMPORTANTE: Recomendado "Public" para GitHub Pages gratis
  5. NO inicialices con README, .gitignore o LICENSE
  6. Haz clic en "Create repository"

Paso 4: Conectar repositorio remoto

  1. GitHub te mostrará instrucciones. Copia y pega en terminal:
    git branch -M main git remote add origin https://github.com/TU_USUARIO/mi-pagina-web.git git push -u origin main
  2. Reemplaza TU_USUARIO con tu usuario de GitHub
  3. Te pedirá autenticación (usa token si es HTTPS)

Paso 5: Activar GitHub Pages

  1. Ve a tu repositorio en GitHub
  2. Abre Settings → Pages
  3. En "Source" selecciona "main" branch
  4. En "Folder" selecciona "/ (root)"
  5. Haz clic en "Save"
  6. GitHub mostrará URL: https://TU_USUARIO.github.io/mi-pagina-web

Paso 6: ¡Listo! 🎉

Tu sitio estará live en ~2 minutos. Accede a la URL mostrada.

💡 Tip: Puedes actualizar el sitio haciendo push en cualquier momento:
git add . git commit -m "Actualización: [descripción]" git push origin main
📱 Publicación en Facebook
ℹ️ Nota: Facebook no "hospeda" tu sitio. Lo que haremos es compartir el enlace en Facebook para que la gente acceda a tu GitHub Pages.

Opción 1: Compartir enlace en Page/Perfil

  1. Ve a tu página o perfil de Facebook
  2. Haz clic en "¿Qué estás pensando?"
  3. Pega la URL: https://tuusuario.github.io/mi-pagina-web
  4. Añade descripción:
    ¡Conoce nuestro nuevo sitio web! Catálogo de productos, panel administrativo y compras integradas. 🛒 👉 Visita: [URL]
  5. Haz clic en "Compartir"

Opción 2: Compartir en página comercial

  1. Ve a tu Página de negocio en Facebook
  2. Abre Publicaciones → Crear
  3. Elige "Enlace" (si quieres que aparezca preview)
  4. Pega la URL
  5. Facebook cargará automáticamente:
    • Imagen (del meta tags si las agregamos)
    • Título del sitio
    • Descripción (del meta tags)
  6. Personaliza el mensaje y publica

Opción 3: Crear anuncio en Facebook Ads (Avanzado)

  1. Ve a Meta Business Suite (facebook.com/business)
  2. Abre Ads Manager
  3. Crea campaña con objetivo "Tráfico al sitio web"
  4. URL de destino: tu sitio GitHub Pages
  5. Personaliza audiencia, presupuesto y plazo

🎨 Mejorar preview en Facebook (Opcional)

Para que la vista previa en Facebook se vea mejor, añade estos meta tags a tu index.html:

<meta property="og:title" content="Mi Asombrosa Tienda Online"> <meta property="og:description" content="Catálogo de productos, gestión administrativa y compras integradas"> <meta property="og:image" content="https://tuusuario.github.io/mi-pagina-web/imagen-preview.jpg"> <meta property="og:url" content="https://tuusuario.github.io/mi-pagina-web"> <meta property="og:type" content="website">
⚠️ Importante: Para que Facebook cargue la imagen en el meta tag, necesita que esté hospedada online (en tu GitHub Pages). Sube una imagen .jpg o .png a la carpeta raíz.

📞 Solución de Problemas

❓ "Mi sitio no se ve bien en GitHub Pages"

Causa: Probablemente un problema de rutas relativas.

Solución: Verifica que todas las rutas sean relativas (../) y no absolutas (/).

❓ "Los estilos de Tailwind no se cargan"

Causa: CDN de Tailwind inaccesible (raro).

Solución: Comprueba tu conexión a internet. Tailwind necesita acceso a cdn.tailwindcss.com

❓ "Las fuentes no cargan"

Causa: Google Fonts inaccesible (muy raro).

Solución: Verifica conexión. Las fuentes son de Google Fonts (100% confiables).

❓ "El sitio funciona localmente pero no en GitHub"

Causa: Archivos no subidos correctamente.

Solución:

git status # Ver qué cambios hay git add . git push origin main # Empujar todos los cambios
